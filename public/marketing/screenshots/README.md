# Marketing screenshots — App Store + Play Store

Generated by `tools/marketing/capture_screenshots.ts`. Reproducible from a
deterministic seed (calm 7-day demo) so re-runs produce the same artifacts
modulo timestamps.

## Layout

```
ios/
  light/   today.png  track.png  goals.png  insights.png  crisis.png   (1290×2796)
  dark/    today.png  track.png  goals.png  insights.png  crisis.png   (1290×2796)
android/
  light/   today.png  track.png  goals.png  insights.png  crisis.png   (1080×2400)
  dark/    today.png  track.png  goals.png  insights.png  crisis.png   (1080×2400)
```

20 PNGs total. The five surfaces are picked to demonstrate the calm-tracking
+ crisis-resources story without surfacing premium gates: home (Today),
drink log + history (Track), goal progress (Goals), insights (Insights),
and the always-on crisis modal.

## Submission workflow

### Apple App Store Connect

App Store Connect requires 6.7" iPhone screenshots (1290×2796) for the
"Display" set. The five `ios/light/*.png` files cover that requirement.
The `ios/dark/*.png` set is optional but improves the listing — submit
them in the same order if Apple's localized listing accepts it.

1. App Store Connect → My Apps → Alchohalt → App Store tab
2. iOS App version → 6.7" Display screenshots → drag the five `ios/light` PNGs
   in surface order: Today, Track, Goals, Insights, Crisis
3. (Optional) Submit `ios/dark` for the alternate localized variant

The 5.5" iPhone slot is no longer required for new apps; skip it.

### Google Play Console

Play Console requires phone screenshots at 1080×2400 (or any 16:9 / 9:16
ratio between 320 and 3840 px, but 1080×2400 is the canonical Pixel 6 size).
The five `android/light/*.png` files cover that requirement.

1. Play Console → Alchohalt → Grow → Store presence → Main store listing
2. Phone screenshots → upload the five `android/light` PNGs in surface order
3. Both light and dark variants can be uploaded under "alternate" assets if
   the listing supports it

## Re-running

```bash
npm install -D playwright tsx
npx playwright install chromium
npm run build
npx vite preview --port 4173 &        # serves the built bundle
npx tsx tools/marketing/capture_screenshots.ts
```

To capture a subset:

```bash
npx tsx tools/marketing/capture_screenshots.ts --surfaces=today,crisis
```

Override the target with `DEV_URL=http://localhost:5173` to point at the
live dev server instead of the preview build.

## Why a 7-day streak (not 366)

Trustworthy listings show realistic state. A user 7 days in is the modal
case for the App Store reviewer's first impression — long enough to read
as "this works," short enough to read as "this could be me." Long streaks
look aspirational and trigger the "promised land" feeling that the calm
voice deliberately avoids.

The seed lives in `makeSeedPayload()` inside the capture script — edit
there to change the demo state for every screenshot at once.
