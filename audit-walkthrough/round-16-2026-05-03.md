## Voice-and-framing checklist

Walked every string added or modified this round:

- [x] No marketing voice (no exclamation, no empty modifiers)
- [x] No "you're broken, this app fixes you" framing
- [x] No clinical jargon the user is asked to apply to themselves
- [x] No false certainty ("always", "never", "guaranteed")
- [x] No second-person commands without context

R16-1 codified the five-question test for milestone surfaces in
`voice-guidelines.md`:
1. Subject of the sentence — observation or gamification?
2. Punctuation — exclamation marks?
3. Verbs — "earn / unlock / win" vs "reach / pass / mark"?
4. Countdown framing — "to go!" vs calendar-fact?
5. Comparative bragging — superlatives vs the number?

- **R16-A** Third arm on the onboarding chip-copy A/B —
  `first-person-trying` ("I'm trying to drink less" / "I'm pausing
  alcohol for now" / "I'm just looking around"). Decouples the two
  changes that R15-B conflated: voice (first-person vs third-person)
  and tone (hedged vs declarative).
- **R16-B** A/B copy variant on the goal-nudge banner. Control
  ("You've been at X std/day this week. Your goal is Y/day. Want to
  revisit it?") vs `softer` ("Your goal is Y/day. This week's been
  around X/day. Some weeks land different — adjust if it's helpful.").
  Both arms preserve R15-2 mitigations: opt-in default-off,
  dismissible 7-day, sober-mode-suppressed.
- **R16-1** Milestone-copy audit. Audited every user-visible
  milestone surface against observation-vs-gamification. Five
  surfaces pass (Milestones.tsx, FirstMonthRibbon, LongTermRibbon,
  BackupAutoVerifyRibbon, internal helpers). One was rewritten:
  `progressCards.tsx` `StreakMilestoneCard` — heading "Streak
  Milestone" → "Current alcohol-free streak", "{n} days to go!" →
  "{n} days from there." The five-question test is codified in
  `voice-guidelines.md`.
- **R16-2** User-installable crisis-line entry. New

  Verified by document-order assertion in the test.
- **R16-3** Date-range export with from/to picker. New `export-range.ts`
