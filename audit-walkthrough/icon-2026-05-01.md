# App icon — design rationale (2026-05-01, R7-A2)

## What shipped

A single-color brand mark: **a horizontal white pause-stroke on a sage-700 ground.**

- Source: `public/branding/icon-source/alchohalt-icon.svg` (full-bleed) +
  `alchohalt-foreground.svg` (Android adaptive foreground, 66% inset).
- Generator: `scripts/gen-app-icons.cjs` (sharp). Run with `npm run gen-icons`.
- Outputs: iOS 1024×1024, Android mdpi → xxxhdpi (full-bleed legacy +
  adaptive foreground), PWA 192/512, web favicon.
- Android background: sage-700 swatch via `ic_launcher_background.xml`.

## Why this mark

### What it isn't

- **Not a stop-sign.** No octagon, no red, no slash. The brand is *pause*, not
  *stop*. Halting alcohol is pausing the relationship — interpreting the name
  as a literal "stop" mark would push the brand toward enforcement, which is
  the wrong voice ("the kind of thing you'd want a quiet friend to say").
- **Not a wordmark / monogram.** "AH" or "A" works at 96px but starts to mush
  at 48px and below — and Android adaptive icons crop to 66% safe area, which
  pushes a typographic mark even smaller. A single stroke survives both.
- **Not a literal glass / bottle / liver.** Anything alcohol-imagery-shaped on
  the launcher screen is a privacy hazard — partners, family, recovery
  sponsors all see launcher icons. The mark must be *ambiguous-but-positive*:
  legible if you know what it is, plausibly some other utility app if you
  don't.

### What it is

- **A horizontal line, centered.** The line is one of the most legible visual
  primitives at small sizes — it survives 16×16 favicon rendering without
  anti-aliasing fuzz that kills typography. Stroke caps are rounded so it
  reads "calm" instead of "surgical".
- **Sage-700 (#3F6055) ground.** Brand action color. Calm, trusted-friend
  green, not aggressive sport green. Matches `--action-primary` in
  `theme.css`.
- **White stroke at ~9% canvas height.** Heavy enough to read at 16×16,
  thin enough at 1024×1024 that it doesn't dominate. Inset to ~52% width
  so there's clear margin on either side.

### Trade-offs the user might want to revisit

- **Visual ambiguity.** A horizontal line is also generic — could be a mute
  toggle, a thinking dash, a horizon. If owner wants more brand specificity,
  options:
  - Add a small dot above or below the stroke (suggests "i" or punctuation).
  - Tilt the stroke 5° (less symmetrical, more "human").
  - Add a 2nd shorter stroke below at 75% width (resembles an "=" but with
    asymmetry).
  Keep the single-stroke version for now because *less is more recoverable* —
  if branding tests show it's too generic, evolving from one stroke is
  cheaper than rolling back from a complicated mark.
- **Greyscale story.** In greyscale, sage-700 → mid-grey (#4F4F4F-ish). The
  white stroke still has 8.6:1 contrast against that, which clears WCAG AA
  for "graphics" (3:1). No greyscale penalty.
- **Dark-mode launcher.** Some Android launchers tint app icons to a uniform
  greyscale in dark themes. The full-bleed sage background protects us
  there because the mark is a high-contrast white-on-dark-grey stroke after
  any tint. iOS launcher does not currently auto-tint.

## What was on the floor before

| Layer | Was | Now |
|---|---|---|
| iOS Assets.xcassets | Empty (Contents.json referenced a missing PNG) | 1024×1024 sage stroke |
| Android adaptive bg | white (`#FFFFFF`) | sage-700 (`#3F6055`) |
| Android adaptive fg | AOSP green Android logo (the placeholder) | white pause-stroke at 66% safe inset |
| Android legacy launcher | (not present at most densities) | full-bleed sage stroke at every density |
| favicon.svg | Bright green (`#22c55e`) "A" in Arial | sage-700 stroke (matches the launcher) |
| pwa-192.png / pwa-512.png | green "A" placeholder | sage stroke matching the launcher |

The previous AOSP-default Android icon was the most embarrassing single
visible artifact — anyone who installed the APK saw "Android Studio sample
project" on their home screen. That's the round-7-blocking item. Done.

## Regen workflow

```bash
# 1. Edit alchohalt-icon.svg / alchohalt-foreground.svg in
#    public/branding/icon-source/.
# 2. Run:
npm run gen-icons
# 3. Commit the PNGs alongside the SVG diff so the source-of-truth and
#    rendered output stay in sync.
```

`sharp` is installed via `npm install --save-dev --no-save sharp` ad-hoc
because we don't want it in `devDependencies` (it's heavy and the script
runs once per icon revision, not per build). The generator script
documents this at the top.

## What this does NOT cover

- **Notification icons.** Android wants a separate small white-on-transparent
  status-bar icon (`drawable/ic_notification.png`). That's a distinct asset
  with different design constraints (single-colored, transparent
  background). Out of scope this round; the Capacitor LocalNotifications
  plugin falls back to the launcher icon, which now at least matches the
  brand. Track separately as a follow-up.
- **Marketing imagery.** Any screenshots / app-store hero / press-kit
  artwork still uses the older brand and may need a refresh against this
  mark. Out of scope — sequence as a marketing pass once owner approves
  the launcher mark.
