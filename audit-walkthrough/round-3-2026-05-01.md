### A2 — AppLock dialog semantics

`AppLock.tsx` (gated off behind `ENABLE_APP_LOCK=false`) had no dialog semantics on the unlock or setup surfaces. With the flag flipped, a screen-reader user would hear "🔒 App Locked" as decorative + the PIN pad as bare buttons. Now:

- Both surfaces are `role="dialog"` `aria-modal="true"` with `aria-labelledby` + `aria-describedby`.
- `useFocusTrap` wraps Tab inside each surface; Escape on setup triggers cancel via `onEscape`.
- Focus auto-moves to the first interactive element on open.
- `PinPad.tsx` (extracted) gives every digit + ✓/← button an `aria-label`. The unlabelled `App Lock` toggle row now carries `aria-pressed` + `aria-label`.
- Error messages render inside `<p role="alert">`.

Refactor side-effect: `AppLock` (102 → 32 lines) and `AppLockSettings` (135 → ~30 lines) both under threshold.

### E — Mobile haptics map

Locked map (documented in `shared/haptics.ts`):

| Event | Haptic | Reason |
|-------|--------|--------|
| `drink-logged` | Light | Confirmation tap, like an iOS keyboard click |
| `af-day-marked` | Light | Same — pressing the Mark AF button is the same action |
| `goal-hit` | Medium | Stronger bump for genuinely good news |
| `milestone-reached` | Medium | Same pulse as goal-hit |
| `error` | Warning | Subtle warning for purchase / sync / IAP failures |
| `crisis-opened` | **silence** (intentional) | Tactile bump in distress = hostile UX. Documented to prevent "completion." |

Wired call sites this round:
- `AlcoholCoachApp.addDrink` → `drink-logged` or `af-day-marked` (volumeMl=0 + abvPct=0 sentinel)
- `AlcoholCoachApp.saveDrink` → `drink-logged` on edit
- `useSubscriptionPurchase` → `error` on purchase failure / exception
- `SyncPanel.setFieldError` → `error` only on `field='form'` (catch-block path); inline validation hints stay haptic-silent so the device doesn't buzz on every typo

**Not yet wired:** `goal-hit` and `milestone-reached`. These need reach-detection in the store update path (compare prior streak/goal status to new on each `addEntry`). Filed for follow-up — the haptic primitive is mapped, the trigger needs the store-level event.

The web build is unchanged (`Capacitor.isNativePlatform()` short-circuits to no-op). For native: requires `npm install @capacitor/haptics` + `ENABLE_NATIVE_HAPTICS=true` + `npx cap sync`, all already in the round-2 commit message.
