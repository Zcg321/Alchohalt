## Round summary

### What landed

- **R19-A**: Voice-drift regression guard. Per-locale banned-phrase
  lexicon for es/fr/de/pl/ru, plus exclamation-mark drift detection
  on calm-voice key prefixes (onboarding, paywall, milestones,
  support, streakBreak, firstMonth, recovery). 7 new tests. Verified
  catches drift by transient injection of "parcours bien-être" into
  fr.json. Catches the exact failure mode R18-3 surfaced manually.
- **R19-B**: Lint cleanup 35 → 30. Real refactors to ErrorBoundary
  (109 → small dispatcher), CrisisResources (130 → 67 lines), and
  TodayPanel (163 → 78 lines). Dev-only files (DevTokensPreview,
  ComponentGallery) get explicit eslint-disable + reason.
  `audit-walkthrough/round-19-lint-triage.md` catalogs the remaining
  30 by category with v1.1 follow-ups.
- **R19-1**: Offline-mode robustness. Scheduler defers sync when
  offline, records "offline; deferred (<reason>)" activity entry,
  auto-retries on browser `online` event. Reason upgrade if multiple
  triggers arrive while offline (manual > mutation > foreground).
  9 new tests. `round-19-offline-audit.md` walks every async path.
- **R19-2**: Battery-impact audit. Confirms zero polling in the
  codebase: 1 setInterval (BreathingTimer, bounded 60s, now
  visibility-aware), 12 one-shot setTimeouts, 2 one-shot rAF sites
  (no loops). Notifications use OS-native scheduling. All
  subscriptions event-driven. BreathingTimer pauses on
  document.hidden so a 5-minute background return doesn't resume
  from "55 seconds elapsed". 1 new test.
- **R19-3**: Storage usage dashboard. New `src/lib/storage/usage.ts`
  wraps two complementary signals — navigator.storage.estimate
  (browser quota) + JSON-serialized app-state size. Soft warning at
  80% on whichever signal trips first. 50 MB app-side soft cap
  matches typical Safari 50-100 MB quota. UTF-8 byte counting via
  TextEncoder for accurate non-ASCII. Surfaces in DiagnosticsAudit
  as a new StorageFieldset. 12 + 2 new tests.
- **R19-4**: Crash reporter wiring. Sentry-compatible envelope
  hand-rolled (no @sentry/browser bundled — saves 50 KB gz and
  strips the default-on breadcrumb capture we don't want). Three
  gates: VITE_SENTRY_DSN at build, settings.crashReportsEnabled at
  runtime, error survives ErrorBoundary. Privacy invariants pinned
  in tests: no breadcrumbs, no user, no tags, no entries / goals /
  notes / settings. New CrashReportsToggle in Settings → Privacy &
  data with explicit "what we send / what we don't" copy. 8 + 3
  new tests.
- **R19-5**: Security researcher judge. Three real findings, all
  fixed: (1) added strict CSP to vercel.json + meta fallback in
  index.html with default-src 'self', script-src 'self', no
  inline-script allowance; (2) extracted the inline loader-removal
  script from index.html into main.tsx; (3) wrapped marked output
  in LegalDocPage with DOMPurify.sanitize() for defense-in-depth.
  Five non-findings audited and clean (cloud-sync E2E encryption,
  npm audit, etc.). Promoted dompurify to direct dep. 2 + 1 new
  tests for CSP-meta presence and no-inline-script invariant.
- **R19-6**: Nineteen-judge spectacular gate. Added security
  researcher (judge 19). 19/19 sign off across all R19 surfaces;
  multiple "Ship hard" verdicts. No cross-judge conflicts.
