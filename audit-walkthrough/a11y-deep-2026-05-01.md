# A11y deep — the things axe doesn't catch (2026-05-01, R7-E)

axe-core covers the rule-checkable layer (color contrast, ARIA shape,
label presence). Round 7's §E asks for the things axe can't tell you:
does color carry information no other signal carries? Does the
heading outline scan? Are skip links actually wired? Are live regions
firing for the right events?

## 4. Live regions

**Existing:**
- `aria-live="polite"` (8 places): offline banner, today panel hero,
  hard-time panel status, sync panel phase, subscription manager
  message, soft-restart banner, **drink-log success (R7-B)**.
- `aria-live="assertive"` (1 place): Toast component.

**Coverage by event class:**

| Event class | Live region exists? | Politeness |
|---|---|---|
| Routine state changes (tab switch, log success) | ✓ R7-B added log-success | polite |
| User-initiated submits (export/import results) | uses `alert()` (browser-modal) | (n/a — modal interrupts) |
| Background state (offline → online) | ✓ | polite |
| Errors (form validation, save failures) | ✓ Toast on render | assertive |
| Sync conflict notifications | ✓ SyncPanel phase changes | polite |

**Gap:** export-import success/failure currently uses
`window.alert()` instead of an aria-live region. Browser-modal works,
but the alert mid-action is a focus-disruption — the user has to
acknowledge the modal even when nothing went wrong. Worth swapping
for an aria-live confirmation in a future round. **Not landing inline
this round** — the modal works correctly, replacement is polish.

## 5. Focus visible

**Today (`src/index.css` line 84):**

```css
*:focus-visible {
  @apply outline-none ring-2 ring-sage-500 ring-offset-2 ring-offset-surface;
  transition: all 0.2s ease-out;
}
```

Global rule. 2px sage ring with 2px offset. Specificity is high
enough that the few `focus:outline-none` overrides scattered around
(TabShell panel container, Toast action button, SyncPanel danger
buttons) don't kill it — `focus:` and `focus-visible:` are different
pseudo-classes; `outline-none` on the former doesn't cancel the
latter.

**Verified by inspection:** every interactive element gets a visible
ring on Tab. Buttons, links, form controls, custom roving-tabindex
tabs (per [A11Y-TABSHELL]).

**Verdict:** consistent. Sage ring works against every background in
both light and dark themes.
