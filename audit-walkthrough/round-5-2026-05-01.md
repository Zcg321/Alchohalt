## Section A — Carry-over (commit `349a690`)

### A1 — Streak transitions resolved

Owner direction was: track active streak vs. longest streak as
separate metrics, never regress longest. The math already separated
them (`computeStreak` for active, `computeLongestStreak` for the
historical max, `computeTotalAFDays` for lifetime cumulative). The
contract is enforced by `getStreakStatus` which never returns the
"0 days" headline for a returning user.

What round 5 added: 8 new test cases in `streak-transitions.test.ts`
covering AF→AF, AF→BREAK, BREAK→AF, BREAK→BREAK, gap→AF, gap→BREAK,
plus invariants (30-day relapse → longest stays at 30; two completed
streaks of 5 then 3 → longest = 5).

What round 5 found and fixed: a real bug. `computeStreak` had no
upper bound — for a user with all-AF history the loop walked back
past year 0 and crashed with `Invalid time value`. For a brand-new
user with `{}` it returned 3650. Fix: bound by earliest-record key
(lex-compare ISO strings) + 3650-iter cap. New users now correctly
return 0.

### A2 — `ai-recommendations.ts` copy pass

Module is gated behind `ENABLE_AI_RECOMMENDATIONS=false` until
owner flips it. Strings rewritten per the round-4 voice guide:

## Section E — Code health (commit `9d03c9e`)

Headline: new global error reporter shim (`src/lib/errorReporter.ts`)
+ test (`errorReporter.test.ts`, 5 cases).

Background: rounds 1–4 had per-tile + top-level `<ErrorBoundary>`
coverage for React render errors. But errors that escape React —
async event handlers, unhandled promise rejections, setTimeout
throws — went silently to the console in production. The reporter
installs `window.addEventListener('error', …)` and
`window.addEventListener('unhandledrejection', …)` and routes them
through a configurable `Reporter` callback.

Default behavior: console-only. **No PII, no network call.** The
native build can flip on a real reporter via `setReporter(fn)`
AFTER explicit user consent — never silently. Wired from
`main.tsx` with a comment that calls out the privacy invariant.

TS strictness: `strict: true` already on. The remaining escape is
`noImplicitAny: false`. 15 sites across `src/store/db.ts` would
need typing; deferred to an incremental pass since the risk
outweighs the round-5 benefit.

Test coverage delta: +20 tests this round (8 streak transitions,
7 SR walkthrough, 5 errorReporter). 538 → 550 pass + 1 skip.

Fixture flake: existing tests sampled. The streak-transitions
file uses `vi.useFakeTimers()` for determinism. Most other tests
using bare `Date.now()` are short-lived assertions where any time
drift is negligible. No further action this round.
