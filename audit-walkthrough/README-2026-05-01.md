## What the audit caught

### 🔴 Material

1. **Fake-analytics PII pump** — `analytics.ts` was silently
   recording drink type, volume, mood, intensity, page views to
   `localStorage` keyed by a generated user-id, with no upload
   destination, no UI to view, no opt-in. Inline comment
   explicitly said "demo purposes." Contradicted the "data stays
   on device" promise. *Gutted in `[…-C]`.*

2. **400 KB of crypto library shipping to every user** —
   libsodium-wrappers-sumo was eager-loaded with the main bundle
   because the static import chain ran from `AlcoholCoachApp` →
   `SettingsTab` → `SettingsPanel` → `SyncPanel` →
   `keys.ts`+`mnemonic.ts` → sodium. Compounded by
   `manualChunks` rule that funneled all node_modules into the
   eager `vendor` chunk. **Eager bundle dropped 568 KB raw / 167
   KB gz.** *Fixed in `[…-E]`.*

3. **Crisis phone numbers were buttons, not links** —
   `<button onClick={callPhone}>` instead of `<a href="tel:…">`.
   Screen readers announced wrong affordance on the most
   sensitive surface in the app. iOS WKWebView lost native
   integrations (long-press preview, Contacts). *Fixed in `[…-D]`.*

4. **`safe-bottom` utility class was undefined** — the iPhone
   bottom-tab bar sat under the home indicator gesture area,
   blocking taps on the rightmost tab. Class was applied; no CSS
   ever defined it. *Fixed in `[…-F]`.*

### 🟠 Moderate

5. **Crisis modal had no focus management** — focus didn't move
   into the dialog on open, didn't return to the trigger on
   close. Keyboard navigation broken. *Fixed in `[…-D]`.*

6. **6 typecheck errors + 3 lint errors landed on the SYNC
   branch** — test fixtures with ghost fields, libsodium typed
   against the slim package that doesn't expose `crypto_hash_sha256`.
   *Fixed in `[…-A]`.*

7. **No eslint guard for direct localStorage** — three files had
   drifted past the Capacitor.Preferences shim (`analytics.ts`,
   `services/theme.ts`, `IconThemeManager.tsx`). *Guard added in
   `[…-C]` so the next drift is a lint error.*

### 🟡 Minor

8. **Decorative SVG icons not aria-hidden** — TabShell, Toast,
   Disclaimer announced "image, Home" / "image, Success" to
   screen readers. *Fixed for high-traffic surfaces in `[…-D]`.*

9. **~1,400 lines of orphan code shipped** — `_deprecated/`
   directory, `QuickActions.tsx + MoodTracker.tsx` chain,
   `IconThemeManager.tsx`. The legacy `MoodTracker` was actively
   writing user mood to localStorage that nothing read.
   *Removed in `[…-B]`.*

## Stamp

I would be proud to ship this for the world to see. The pre-audit
surface had a leak the user wouldn't see (analytics PII), a perf
regression the user would feel (839 KB eager bundle), and an a11y
miss on the most sensitive surface in the app (Crisis
buttons-not-links). All three are fixed. The remaining items are
owner-eye copy decisions and refactors that deserve their own
focused work.
